Design Notes

tier_at_booking on Registration captures the student's tier at the moment they booked. The student's global tier on the Student table can change anytime, but the tier used for pricing is the one at booking time. This also serves as the income history — no separate tracking needed.

settings_locked on Class flips to true when the first Registration is created. After that, economic fields (room_cost, min_rate, target_rate, min_students, max_students) become immutable.

WaitlistEntry is separate from Registration to cleanly model the hybrid promotion rules. When promoted, a Registration is created and linked via registration_id.

StudioClass is intentionally disconnected from Room and Student. It's a simple log entry for the teacher's calendar and income reporting.

Notification uses a polymorphic recipient (teacher or student) so both share the same inbox infrastructure.

rental_rate on TeacherRoom is private to each teacher — never shown to other teachers using the same room.
